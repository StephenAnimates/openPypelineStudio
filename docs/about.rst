=====
About
=====

**openPypeline Studio** is the modernized, Python-based successor to the original **openPipeline** framework, an open-source framework for managing animation production data and workflow.

History & Origins
=================
First drafted as the *openPipeline Specification v0.1* in August 2007, its first implementation was a MEL-based plug-in for Autodesk Maya that handled specific aspects of production: automatic directory structures, file naming conventions, revision control, and modularity that makes multi-artist workflows possible. Versions for other production applications (like Nuke, Houdini, and Unreal Engine) are actively in development by the community.
